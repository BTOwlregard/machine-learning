Implementation: Outlier Detection¶

Detecting outliers in the data is extremely important in the data preprocessing step of any analysis. The presence of outliers can often skew results which take into consideration these data points. There are many "rules of thumb" for what constitutes an outlier in a dataset. Here, we will use Tukey's Method for identfying outliers: An outlier step is calculated as 1.5 times the interquartile range (IQR). A data point with a feature that is beyond an outlier step outside of the IQR for that feature is considered abnormal.

In the code block below, you will need to implement the following:

  • Assign the value of the 25th percentile for the given feature to Q1. Use np.percentile for this.
  • Assign the value of the 75th percentile for the given feature to Q3. Again, use np.percentile.
  • Assign the calculation of an outlier step for the given feature to step.
  • Optionally remove data points from the dataset by adding indices to the outliers list.

NOTE: If you choose to remove any outliers, ensure that the sample data does not contain any of these points!
Once you have performed this implementation, the dataset will be stored in the variable good_data.

Answer: The first two principal components explain 70.68% of the variance in the overall feature set and the first four together explain 93.11% of the variance.

Each principal components captures a customer spend pattern - a ratio of relative spending among categories - that is present in parts of the data. For example, the first principal component identifies a pattern of spending whereby: milk and gorcery spending move together, detergent_paper is nearly 2x grocery or milk, deli spending is positively correlated with these features but at a lower level of spend (<.5x grocery or milk spending), and fresh and frozen spending is inversely related to grocery and milk.

The first and second principal components together indicate that customers will tend to spend heavily on milk, grocery and detergents_paper together or on fresh, frozen, and deli together. If they spend heavily in one of the three categories in those groups, they likely spend heavily on the other two as well. The first two principal components could be thought of, in this instance, as features representing a weighted basket of goods (I note that negative weights complicate the analogy, but I still find it useful.) If all we knew about these customers was how much they spent in weighted basket (principal component) 1 and weighted basket 2, we would still be able to capture ~70% of the variability in the original feature set.

The first and second component explain much of the variance in the data, but component 2 overgeneralizes the relationship between fresh, frozen and deli. The third and fourth principal components largely serve to invert some of the relationships among fresh, frozen, and deli, which would allow for finer distinctions among customer spending habits. That is, the second principal component indicates that these three categories are often positively correlated; the third principal component indicates further that there are exception to this rule - a subset of customers for whom frozen and deli spend is inversely related to frozen. Component 4 similarly indicates that there are customers who spend a lot on frozen, but not on deli and fresh.

The fifth and sixth components indicate that there are also exceptions to the positive correlation between milk, grocery and detergents_paper, but these distinctions are less prominent; we can capture ~93% of the variance in the data without recognizing them.

Question 6¶

What are the advantages to using a K-Means clustering algorithm? What are the advantages to using a Gaussian Mixture Model clustering algorithm? Given your observations about the wholesale customer data so far, which of the two algorithms will you use and why?

Answer: The primary advantage of a K-Means clustering algorithm, over GMM, is that K-Means definitively assigns each data point to a cluster. GMM, by comparison, assigns to each data point a probability of its belonging to each cluster (wherein each cluster is modeled as a multivariate Gaussian distribution). Moreover, while K-means imposes spherical clusters by basing cluster assignments on Euclidian distance, GMM allows for ellipsoid clusters. That is, GMM does not place contraints on the covariance structure of its clusters, whereas K-Means necessarily produces clusters with 0 cluster covariance (clusters will thus be [hyper-]spherical).

In this instance, I see no reason not to use GMM. Given the greater flexibility of cluster shape and no imminent need for deterministic clusters (we can always assign points to clusters based on the highest cluster probability in GMM), GMM may give us a more informative result.

Depending on the problem, the number of clusters that you expect to be in the data may already be known. When the number of clusters is not known a priori, there is no guarantee that a given number of clusters best segments the data, since it is unclear what structure exists in the data if any. However, we can quantify the "goodness" of a clustering by calculating each data point's silhouette coefficient. The silhouette coefficient for a data point measures how similar it is to its assigned cluster from -1 (dissimilar) to 1 (similar). Calculating the mean silhouette coefficient provides for a simple scoring method of a given clustering.

Question 10¶

Companies will often run A/B tests when making small changes to their products or services to determine whether making that change will affect its customers positively or negatively. The wholesale distributor is considering changing its delivery service from currently 5 days a week to 3 days a week. However, the distributor will only make this change in delivery service for customers that react positively. How can the wholesale distributor use the customer segments to determine which customers, if any, would react positively to the change in delivery service?
Hint: Can we assume the change affects all customers equally? How can we determine which group of customers it affects the most?

Answer: If we operate on the assumption that customers in a similar line of business or with similar buying habits will respond similarly to the delivery service change, it makes sense to ensure that the segment they test the delivery service change on is representative of the range of customer segments. That is, we have identified two distinct customer segments in the data. If they run an A/B test (some customers get new 3-day delivery service, while some remain on 5-day service), both the test and control groups should contain a mix of customers in Cluster 1 and 0, in rough proportion with how many are in each cluster (~56% should come from Cluster 1). This structure (single test group; single control group) assumes that we are able to collect customer-level response data and evaluate test-vs-control effects for each of the clusters separately after running the test. If we will only receive aggregated group-level response data, we will need to instead run separate tests for each cluster in order to determine within-cluster response effects. That is, we will need a separate test and control group for Cluster 0, and another test and control group for Cluster 1 (total of four groups) to ensure that we can identify cluster-specific behavior effects. The ultimate aim of the A/B test would then be to generalize the response of the test population from each cluster -- or rather the difference in response between test and control for that cluster -- to the entire cluster and implement the new delivery schedule on a cluster-specific basis.
